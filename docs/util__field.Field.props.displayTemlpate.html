displayTemplate

property
{Renderer}
util/field.Field.props.displayTemlpate  

Renderer

Formats the field into a renderer in the list and details view of the data-admin component. The renderer has the scope of the list-table or property table. The simplest displayTemplate value would be the default, which is object[field.name]. (make sure to surround values with brackets)

In this example, the scope of the table components provide access to each row as object and the current field as field.

In addition, other properties can be accessed and combined by providing it ``. Custom helpers and other methods may also be registered and utilized. For instance, if we created a global helper capitalize(property) we could access it with capitalize object.prop_name.

For a local helper, an additional method could be added to the field, like

{
name: 'prop',
alias: 'Property',
capitalize: function(val){
    return val.toUpperCase();
}
}

In a stache template, this could be rendered using field.capitalize(object.prop)
